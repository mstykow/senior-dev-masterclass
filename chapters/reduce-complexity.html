"Software architecture is overrated, clear and simple design is underrated"

Gergerly Orosz

  • From "The Pragmatic Engineer" blog

  • Don't go looking for nails that you can hit with an architecture
  • The first goal of designing a system is simplicity
  • Clean design is similar to clean code

Complexity Defined

"Complexity is anything related to the structure of a software system that makes it hard to understand and modify the system."

John Ousterhout

  • Cyclomatic complexity: number of linearly independent paths with a piece of source code (Thomas McCabe, 1976)
  • Cyclomatic complexity is not perfect, but it can tell you how many unit tests you'd need to test the function

Symptoms of Complexity

Change Amplification

Simple functionality changes require code modifications in many different places

Cognitive Load

How much do I need to understand about the system as a whole to make a change?

Unknown Unknowns

Things you must know to complete your task but have no way of knowing

  • Change amplification example: Redux
  • Cognitive load example: useEffect
  • More lines of code can be better than fewer lines of code if cognitive load is reduced (for-loop vs. reduce)
  • Unknown unknowns example: how many people are still using that deprecated endpoint I wish I had built monitoring for? when i change the global background color, which nested pages have local overrides that will bite me?

Causes of Complexity

Dependencies

Obscurity

  • Essential vs. accidental complexity
  • A dependency exists when a given piece of code cannot be understood and modified in isolation
  • You cannot eliminate dependencies: the question is how you manage them (e.g. cyclic dependencies)
  • Obscurity occurs when important information is not obvious: lack of specificity, documentation, types, ...

Working Code Isn't Enough

The first step towards becoming a good software designer is to realize that working code isn't enough.

It's not acceptable to introduce unnecessary complexities in order to finish your current task faster.

Most of the code in any system is written by extending the existing code base, so your most important job as a developer is to facilitate those future extensions.

Your job as a developer is not just to create code that you can work with easily, but to create code that others can also work with easily.

  • strategic: your primary goal is to produce a great design, which also happens to work
  • investment mindset: learn the joy of delayed gratification

Beware the Tactical Tornado

How to Measure Code Quality

  1. What is your code churn?
  2. Can your code be easily read, even by beginner developers?
  3. Is your code extensible, can it be edited by developers who aren't the original author?
  4. Is your code easy to maintain?
  5. How's the code for portability?
  6. Is your code well tested for quality and bugs (does it work)?
  • Misuse of DRY and "clean code" principles to conceal complexity with abstractions and in reality make the code even more complex.
